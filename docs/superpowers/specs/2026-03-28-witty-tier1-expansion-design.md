# Design: Tier 1 "가격" 재해석 키워드 확장

## 목표

현재 ~80개인 Tier 1 위트형 키워드를 ~150개로 확장. "○○의 가격"을 창의적으로 재해석하여 사용자에게 "딱 맞는 곳이네!" 경험을 제공.

## 범위

- 50~100개의 새 Tier 1 키워드-URL 매핑 추가
- 생활 밀착형 + 추상적 개념형 + 직업/연봉 복합 구성
- 모든 URL은 웹검색으로 실존/정확성 검증
- 구현 전 기존 키워드 충돌 확인 필수 (`npx tsx generate-top1000-tsv.ts 2>&1 | grep SKIP`)

## 기존 키워드 충돌 주의

아래 키워드는 이미 TIER1_KEYWORDS 또는 CATEGORIES에 존재하므로 **추가하지 않는다:**
- `수능`, `여행`, `병원`, `운전면허`, `결혼반지`, `이사`, `강아지사료`, `고양이간식`
- 구현 시 스크립트를 `--force` 없이 실행하여 SKIP 목록으로 충돌을 사전 확인할 것

## 카테고리

### A. 생활 밀착형 "가격" 재해석

일상에서 "이거 얼마야?"라고 물을 법한 상황들. 한 단계 추론("키워드→가격")이 명확해야 한다.

**인생 이벤트 비용:**
- 결혼, 출산, 장례, 이혼, 입양, 돌잔치비용
- "결혼.가격.kr" → 결혼비용 통계/계산기

**의료/미용 시술비:**
- 치과, 교정, 라식, 성형, 타투, 필러, 보톡스, 임플란트, 탈모치료
- "라식.가격.kr" → 라식 가격 비교 사이트

**자격증/시험 응시료:**
- 한국사, 자격증, 공인중개사
- "자격증.가격.kr" → 자격증 응시료 안내

**공공요금/세금:**
- 군대(병사봉급), 국회의원(의원연봉), 과태료, 주차위반, 음주운전벌금
- "군대.가격.kr" → 병사 봉급표

**반려동물:**
- 강아지입양, 고양이입양, 동물병원
- "동물병원.가격.kr" → 동물의료비 가이드

### B. 직업/연봉 시리즈 (신규)

"이 직업의 가격(연봉)은?" — 가장 자연스러운 "가격" 재해석.

- 의사, 변호사, 판사, 교사, 소방관, 경찰, 파일럿, 약사, 간호사, 회계사
- "의사.가격.kr" → 직종별 평균연봉 (work.go.kr 또는 saramin.co.kr)

### C. 추상적 개념 "가격" 재해석

"이것의 가격이 있다면?"이라는 위트. **한 단계 추론**이 가능한 것만 포함.

**시간/노동:**
- 시간(최저시급), 야근(야근수당), 알바(시급검색), 퇴직(퇴직금계산기)
- "시간.가격.kr" → 최저임금위원회

**법적 대가:**
- 실수(과태료), 명예(명예훼손벌금)
- "실수.가격.kr" → 경찰청 범칙금 조회

**한국 문화:**
- 세뱃돈(시세), 추석(선물세트가격), 축의금(축의금시세)
- "세뱃돈.가격.kr" → 세뱃돈 시세 통계

**기타 위트:**
- 명품(리세일시세 → kream.co.kr), 행복(세계행복지수), 치킨집창업(공정위 정보공개서)
- "명품.가격.kr" → KREAM 리세일 시세

### 제외된 키워드 (리뷰 결과)

2단 추론이 필요하거나 타겟 URL이 불명확한 항목들:
- ~~사랑→결혼비용~~ (사랑→결혼→비용, 2단 추론)
- ~~꿈→창업비용~~ (꿈→창업→비용, 2단 추론)
- ~~자유→이민비용~~ (한국어 랜딩페이지 부재)
- ~~거짓말→사기죄~~ (검색 의도 불분명)
- ~~비밀→NDA위약금~~ (B2B 맥락, 일반 사용자 대상 아님)

### URL 선정 기준
1. "이 키워드의 가격을 알고 싶어서 왔다"는 사용자 의도에 가장 부합하는 사이트
2. 가능한 한 가격/비용 정보 페이지에 직링크. 단, 정부 사이트의 딥링크는 리디자인 시 깨지므로 안정적인 상위 페이지도 허용
3. 한국어 사이트 우선
4. 정부/공공기관 > 전문비교사이트 > 포털검색

### URL 검증
- 모든 URL을 웹검색으로 실존 여부 확인
- whitelist.json 도메인 추가 시 신뢰할 수 있는 사이트만

### 품질 기준
- blocklist/profanity 통과 필수
- "가격" 재해석이 **한 단계 추론**으로 연결될 것 (억지스럽지 않은 자연스러운 연결)
- URL이 실제로 가격/비용 정보를 보여줄 것

### 파이프라인

**주의: `--force` 플래그를 seed-data.ts와 함께 사용하면 안 됨!**
`--force`는 기존 키워드도 포함하여 TSV를 출력하므로, seed-data.ts가 모든 기존 파일의 `created` 날짜를 오늘로 덮어쓰게 된다.

```bash
# 1. 스크립트에 키워드 추가 후 (--force 없이 실행 → 새 키워드만 출력)
npx tsx scripts/generate-top1000-tsv.ts > scripts/top1000-keywords.tsv
# 2. JSON 파일 생성 (새 키워드만 생성됨)
npx tsx scripts/seed-data.ts scripts/top1000-keywords.tsv ./data
# 3. TSV 전체 아카이브 별도 생성 (문서화 용도, seed-data에 연결하지 않음)
npx tsx scripts/generate-top1000-tsv.ts --force > /tmp/full-keywords-archive.tsv
cp /tmp/full-keywords-archive.tsv scripts/top1000-keywords.tsv
# 4. 테스트
npm test
```

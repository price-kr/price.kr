# 📄 Product Requirements Document (PRD): 집단 지성 오픈소스 단축 URL 서비스 — 가격.kr

## 1. 프로젝트 개요 (Overview)
본 프로젝트는 사용자가 브라우저 주소창에 `단어.가격.kr`(예: `만두.가격.kr`)을 입력하면, **Cloudflare 와일드카드 DNS + Workers** 구조를 통해 커뮤니티가 투표하여 선정한 최적의 목적지(최저가 쇼핑몰, 트렌드 페이지 등)로 리다이렉트해 주는 서비스입니다.

**아키텍처:** Cloudflare (리다이렉트 엔진) + Vercel Free (웹 앱) 하이브리드 구조

**동작 방식:**
1. `만두.가격.kr` 접속 → Cloudflare 와일드카드 DNS가 Cloudflare 프록시로 라우팅
2. Cloudflare Workers가 `Host` 헤더에서 서브도메인(`만두`)을 추출
3. Cloudflare KV에서 키워드를 조회하여 **최종 목적지 URL로 직접 리다이렉트** (1회 리다이렉트로 완료)
4. 매핑이 없는 키워드의 경우, `가격.kr/{keyword}` 페이지로 302 리다이렉트하여 Vercel에서 호스팅하는 검색/제안 페이지를 표시

> **참고:** DNS는 리다이렉트를 수행하지 않습니다. 와일드카드 DNS 레코드는 모든 `*.가격.kr` 요청을 Cloudflare 프록시로 라우팅할 뿐이며, 실제 리다이렉트는 Cloudflare Workers에서 처리합니다.

> **왜 하이브리드 구조인가:** Vercel Free 플랜은 와일드카드 도메인을 지원하지 않습니다(Pro 이상 필요). Cloudflare는 무료 플랜에서 와일드카드 DNS + SSL + Workers를 모두 제공하므로, 리다이렉트 엔진은 Cloudflare에, 웹 앱은 Vercel에 배치하여 **$0/월** 운영을 실현합니다.

중앙 집중형 관리 방식을 탈피하여, GitHub의 Issue와 PR(Pull Request), 그리고 커뮤니티 투표(Emoji)를 통해 100만 개의 한국어 키워드 매핑 데이터를 자동으로 구축하고 유지보수하는 완전 오픈소스 지향 파이프라인을 구축하는 것을 목표로 합니다.

## 2. 핵심 목표 (Objectives)
* **초고속 리다이렉트:** 엣지(Edge) 네트워크를 활용하여 사용자가 지연(Latency) 없이 목적지에 도달하도록 합니다. 목표: p99 응답 시간 200ms 이내.
* **100만 개 키워드 확장성:** Git 저장소의 병합 충돌(Merge Conflict)을 방지하고 성능 저하가 없는 최적의 데이터 디렉토리 구조를 설계합니다.
* **유지보수 제로(0) 자동화:** 관리자의 개입 없이, 사용자의 제안과 커뮤니티의 투표만으로 시스템이 스스로 업데이트되는 생태계를 구축합니다.

## 3. 타겟 사용자 (Target Audience)
* **엔드 유저 (End User):** 특정 상품의 최저가를 찾거나 특정 키워드에 대한 가장 유용한 링크에 빠르게 접속하고 싶은 일반 사용자.
* **기여자 (Contributor):** 유용한 링크를 공유하고 싶거나, 오픈소스 생태계에 Issue 작성 및 투표로 가볍게 참여하고 싶은 커뮤니티 멤버.

## 4. 기술 스택 (Tech Stack)

### 리다이렉트 엔진 (Cloudflare — 무료)
* **DNS:** Cloudflare 와일드카드 DNS (`*.가격.kr` → Cloudflare 프록시로 라우팅)
* **SSL/TLS:** Cloudflare Universal SSL — 와일드카드 인증서 (`*.가격.kr`) 무료 자동 발급. IDN(한글) 도메인은 퓨니코드(`xn--o39a88s.kr`)로 등록되며, 배포 전 인증서 발급이 정상 동작하는지 사전 검증 필요
* **라우팅 처리:** Cloudflare Workers — `Host` 헤더에서 서브도메인 추출 → 퓨니코드 디코딩 → Cloudflare KV 조회 → 최종 URL로 302 리다이렉트
* **캐시 및 스토리지:** Cloudflare KV (무료: 일일 100K 읽기, 1K 쓰기, 1GB 저장)
* **배포 도구:** Wrangler CLI (`wrangler deploy`)

### 웹 앱 (Vercel — Free 플랜)
* **프레임워크 & 호스팅:** Next.js, Vercel Free 플랜
* **역할:** 메인 검색 UI (`가격.kr`), 미등록 키워드 안내 페이지 (`가격.kr/{keyword}`), 정적 페이지 (개인정보처리방침 등)

### 공통
* **데이터 버전 관리:** GitHub Repository (JSON 파일 기반)
* **CI/CD 및 자동화:** GitHub Actions (Issue 파싱, PR 자동 생성, 투표 집계, 병합, Cloudflare KV 동기화)
* **모니터링:** Vercel Web Analytics (무료 기본 제공) + Cloudflare Workers Analytics (무료) + 커스텀 대시보드 (리다이렉트 성공/실패율, 인기 키워드, 응답 시간 — Cloudflare Workers에서 자체 집계)

## 5. 핵심 기능 (Core Features)

### 5.1. 와일드카드 DNS + Cloudflare Workers 라우팅

#### DNS 라우팅 (Cloudflare)
* Cloudflare에 `*.가격.kr` 와일드카드 DNS를 설정하여 모든 서브도메인 요청을 Cloudflare 프록시로 라우팅합니다.
* 한글 도메인(`가격.kr`)은 KISA를 통한 `.kr` 한글 도메인 등록이 필요합니다.
* Cloudflare에는 퓨니코드(`xn--o39a88s.kr`)로 도메인을 등록합니다.

#### Cloudflare Workers 처리 (리다이렉트 전담)
* Cloudflare Workers에서 요청의 `Host` 헤더를 파싱하여 서브도메인을 추출합니다.
* 한글 서브도메인의 경우 브라우저가 전송하는 **퓨니코드(Punycode)**를 디코딩하여 원래 한글 키워드로 변환합니다.
  * 퓨니코드 변환은 배포 전 실제 값을 검증해야 합니다. (예: `punycode.toUnicode()` 사용)
* Cloudflare KV에서 키워드를 조회하여 **최종 목적지 URL로 `302 Redirect`** 를 수행합니다.
  * 예: `만두.가격.kr` → (Workers에서 KV 조회) → 네이버 쇼핑 최저가 페이지로 직접 이동
* 매핑이 없는 키워드의 경우, `가격.kr/{keyword}` 페이지로 302 리다이렉트하여 Vercel에서 호스팅하는 검색/제안 페이지를 표시합니다.
* 서브도메인이 없는 요청(`가격.kr` 자체)은 Cloudflare DNS에서 Vercel로 직접 라우팅합니다 (Workers를 거치지 않음).

#### Vercel 웹 앱 (Next.js)
* Vercel Free 플랜에서 `가격.kr` 도메인으로 Next.js 앱을 호스팅합니다.
* `app/[keyword]/page.tsx`: 미등록 키워드 안내 + 유사 키워드 추천 + '새 단어 제안하기' 버튼
* `app/page.tsx`: 메인 검색 UI

> **302 Redirect를 사용하는 이유:** 커뮤니티 투표에 의해 목적지 URL이 변경될 수 있으므로, 브라우저가 영구적으로 캐싱하는 301 대신 302를 사용합니다. 응답 헤더에 `Cache-Control: private, no-store`를 설정하여 브라우저/CDN이 리다이렉트를 캐싱하지 않도록 합니다.

> **뒤로 가기(Back) 문제 대응:** 리다이렉트 후 사용자가 뒤로 가기를 누르면 다시 리다이렉트되는 문제가 발생할 수 있습니다. 이를 방지하기 위해 `history.replaceState`를 활용하거나, 짧은 중간 페이지(리다이렉트 안내 + 자동 이동)를 두는 방안을 Phase 2에서 검토합니다.

### 5.2. 확장 가능한 100만 개 데이터 구조
Git의 파일 개수 및 용량 제한, PR 충돌을 피하기 위해 **초성 폴더 > 키워드당 1파일** 구조로 JSON을 분산 저장합니다.
```text
📦 data
 ┣ 📂 ㄱ (초성)
 ┃ ┣ 📂 가 (첫 글자)
 ┃ ┃ ┣ 📜 가방.json  # { "keyword": "가방", "url": "...", "created": "..." }
 ┃ ┃ ┗ 📜 가구.json
 ┃ ┗ 📂 고 (첫 글자)
 ┃   ┗ 📜 고기.json
 ┣ 📂 ㅁ (초성)
 ┃ ┣ 📂 마 (첫 글자)
 ┃ ┃ ┗ 📜 마늘.json
 ┃ ┗ 📂 만 (첫 글자)
 ┃   ┗ 📜 만두.json
 ┣ 📂 _en (영문 키워드)
 ┃ ┗ 📜 iphone.json
 ┣ 📂 _num (숫자 키워드)
 ┃ ┗ 📜 3M.json
```

> **키워드당 1파일 구조의 이유:** 동일 파일을 수정하는 복수 PR 간 병합 충돌(merge conflict)을 원천적으로 방지합니다. 디렉토리를 3단계(초성 > 첫 글자 > 키워드 파일)로 분산하여 단일 디렉토리에 파일이 과도하게 집중되는 것을 방지합니다. 영문/숫자 키워드는 별도 폴더(`_en`, `_num`)로 관리합니다.

> **Git 100만 파일 관리 주의사항:**
> * `git clone`이 수십 분 소요될 수 있으므로, 기여자 가이드에 **sparse checkout** 사용법을 필수 안내
> * GitHub Actions에서는 `actions/checkout`에 **partial clone** (`filter: blob:none`) 옵션 적용
> * GitHub 웹 UI에서는 디렉토리당 1,000개 이상 파일 탐색이 불가하므로 3단계 디렉토리로 분산 필수

### 5.3. 커뮤니티 기반 자동화 파이프라인 (The Auto-Merge Flow)
1. **제안 (Issue):** 사용자가 정해진 양식의 GitHub Issue(키워드, 타겟 URL)를 생성합니다.
2. **유효성 검증:** GitHub Actions 봇이 Issue를 읽고 다음을 검증합니다:
   * URL이 허용 도메인 화이트리스트에 포함되어 있는지
   * 키워드가 금지 목록(상표명 등)에 해당하지 않는지 — 유사 문자/띄어쓰기 변형도 검사 (정규화 후 비교)
   * 동일 키워드에 대한 기존 매핑이 있는지 (있다면 "변경 제안"으로 분류)
   * Google Safe Browsing Lookup API를 통한 URL 안전성 검증 (무료 비상업적 용도, 일일 10,000건 제한 — Phase 3에서 Update API로 전환 검토)
3. **PR 자동 생성:** 유효성 검증을 통과하면, 해당 키워드의 JSON 파일을 수정하는 PR을 자동으로 엽니다.
4. **투표 (Voting):** 커뮤니티가 해당 PR에 👍(찬성) 또는 👎(반대) 이모지를 남깁니다.
   * 투표 자격: GitHub 계정 가입 후 30일 이상 경과한 계정
   * 동일 키워드에 복수 URL 제안이 있을 경우, 가장 높은 찬성 수를 받은 PR이 우선 병합됩니다.
5. **자동 병합 (Auto-Merge):** 정해진 조건을 충족하면 봇이 `main` 브랜치에 PR을 병합합니다.
   * 병합 조건: 찬성 ≥ **동적 임계값** (초기 3개 → 커뮤니티 성장에 따라 5개, 10개로 상향), 찬성 > 반대
   * 병합 전 자동 rebase를 수행하여 충돌 방지
6. **Cloudflare KV 동기화:** `main` 브랜치 업데이트를 감지하여, 변경된 키워드를 Cloudflare KV에 반영합니다 (Cloudflare API 사용).
   * 동기화 실패 시 재시도 로직(최대 3회) + Slack/Discord 알림
   * 수동 동기화 트리거 제공 (`workflow_dispatch`)

### 5.4. 기존 매핑 수정 및 삭제

* **수정 제안:** "키워드 변경 요청" Issue 양식을 통해 기존 매핑의 목적지 URL 변경을 제안할 수 있습니다. 신규 등록과 동일한 투표 프로세스를 거칩니다.
* **삭제 요청:** 링크가 깨진 경우(dead link), 저작권/상표권 침해 신고, 악성 사이트로 변질된 경우 등을 사유로 삭제를 요청할 수 있습니다. 긴급 삭제(피싱/악성)는 관리자 단독 승인으로 즉시 처리합니다.
* **자동 URL 헬스체크:** GitHub Actions 스케줄(주 1회)로 등록된 URL의 유효성을 배치 검사하고, 접근 불가 URL에 대해 자동으로 "수정 필요" Issue를 생성합니다.

## 6. 사용자 플로우 (User Flow)

* **데스크톱 링크 사용자:** 주소창에 `만두.가격.kr` 입력 ➔ Cloudflare 와일드카드 DNS로 Cloudflare 프록시 도달 ➔ Workers가 서브도메인 추출 후 Cloudflare KV 조회 ➔ 네이버 쇼핑 최저가 페이지로 직접 이동
* **모바일 사용자:** 모바일 브라우저 주소창에 한글 서브도메인을 직접 입력하면 검색으로 처리될 수 있으므로, `가격.kr` 웹사이트의 **검색 UI**를 주요 진입 경로로 제공합니다. Phase 3에서 앱 딥링크를 추가 지원합니다.
* **키워드 미등록 시:** `새단어.가격.kr` 입력 ➔ Workers에서 KV 조회 실패 ➔ `가격.kr/새단어` 페이지로 302 리다이렉트 ➔ Vercel 웹 앱에서 유사 키워드 추천 + "이 단어 제안하기" 버튼 표시
* **데이터 기여자:** GitHub 방문 ➔ '새 단어 제안하기' Issue 작성 ➔ 봇이 유효성 검증 후 PR 생성 ➔ 커뮤니티 투표 진행 ➔ 통과 시 실제 서비스에 반영
* **에러 발생 시:** Cloudflare KV 장애 등으로 조회 실패 ➔ Workers에서 "일시적인 오류가 발생했습니다. 잠시 후 다시 시도해 주세요." 응답 표시 + `가격.kr` 메인 페이지로의 링크 제공

## 7. 거버넌스 정책 (Governance)

### 7.1. 관리자 정의
* **초기 관리자:** 프로젝트 창립 멤버 (GitHub Organization Owner)
* **관리자 권한:** 긴급 삭제, 화이트리스트 수정, 금지 키워드 관리, 투표 임계값 조정
* **관리자 추가:** 기존 관리자 2인 이상의 동의로 새 관리자 추가

### 7.2. 키워드 충돌 해결
* 동일 키워드에 대해 복수 URL이 제안된 경우, 투표 기간(7일) 동안 가장 높은 순찬성(찬성 - 반대)을 받은 PR이 병합됩니다.
* 이미 등록된 키워드에 대한 변경 제안은 "변경 제안" 라벨이 부착되며, 기존 등록보다 높은 임계값(기본 임계값 × 1.5)이 적용됩니다.
* **분쟁 에스컬레이션:** 투표 결과에 이의가 있는 경우, "분쟁 해결" Issue를 통해 관리자에게 중재를 요청할 수 있습니다. 최종 결정권은 관리자에게 있습니다.

### 7.3. 금지 키워드 정책
* 등록 상표명(쿠팡, 삼성, 네이버 등)은 금지 목록으로 관리하며, 자동 검증 단계에서 차단됩니다.
* **우회 방지:** 키워드를 정규화(공백 제거, 특수문자 제거, 유니코드 정규화)한 후 금지 목록과 비교합니다. (예: "쿠.팡", "쿠팡쿠팡", "ㅋㅍ" 등 변형 차단)
* 상표권자의 이의 제기(takedown request) 처리 절차를 운영합니다.
* 비속어, 혐오 표현 등은 금지 키워드 필터로 차단합니다.

### 7.4. 허용 도메인 화이트리스트
초기에는 신뢰할 수 있는 도메인만 허용하여 피싱/악성 사이트 등록을 방지합니다:
* 네이버 쇼핑, 쿠팡, 11번가, G마켓 등 주요 쇼핑몰
* Google, Naver, Daum 등 검색엔진
* 위키백과 등 공공 정보 사이트
* 화이트리스트 추가/삭제는 별도의 PR + **관리자 승인**으로 진행
* **주의:** 화이트리스트 도메인 내에서도 특정 URL이 사기/피싱 페이지일 수 있으므로, Google Safe Browsing API를 통한 개별 URL 검증을 병행합니다.

## 8. 비기능 요구사항 (Non-Functional Requirements)

### 8.1. 성능
* 리다이렉트 응답 시간: p99 < 200ms (한국 리전 기준)
* Cloudflare KV 조회: 평균 < 10ms (Cloudflare 글로벌 엣지 네트워크)

### 8.2. 가용성
* 목표 SLA: 99.9% uptime
* Cloudflare KV 장애 시 fallback: Workers에서 GitHub Raw Content URL(`raw.githubusercontent.com`)을 통해 JSON 파일을 fetch하여 조회합니다. (레이턴시 증가 허용, 요청된 키워드 파일만 개별 조회). **주의:** GitHub Raw Content는 비인증 시 시간당 60회 rate limit이 있으므로, 인증 토큰(GitHub Token)을 사용하여 시간당 5,000회로 확장하고, 추가로 응답을 Workers의 Cache API로 캐시하여 중복 요청을 줄입니다.

### 8.3. 보안
* HTTPS 전용 (HSTS 적용) — Cloudflare에서 자동 적용
* 와일드카드 SSL 인증서 자동 갱신 — Cloudflare Universal SSL (무료)
* DDoS 방어: Cloudflare 무료 플랜 기본 제공 (Layer 3/4/7)
* Rate Limiting: Cloudflare Workers 내에서 자체 구현 (IP 기반 분당 60회 제한). Cloudflare의 유료 Rate Limiting 규칙은 사용하지 않고, Workers 코드 내에서 KV 또는 인메모리 카운터로 구현
* 입력 검증: 서브도메인에 대한 XSS/인젝션 공격 방어

### 8.4. 개인정보보호
* 리다이렉트 과정에서 개인 식별 정보(PII)를 수집하지 않습니다.
* 통계 목적의 접속 로그는 IP를 익명화하여 저장합니다.
* 개인정보처리방침 페이지를 `가격.kr/privacy`에 게시합니다.

## 13. 용어 정의 (Glossary)

| 용어 | 설명 |
|------|------|
| **와일드카드 DNS** | `*.가격.kr` 형태로 모든 서브도메인을 하나의 서버로 라우팅하는 DNS 설정 |
| **퓨니코드 (Punycode)** | 한글 등 비ASCII 문자를 ASCII로 인코딩하는 표준 (RFC 3492). 실제 변환값은 구현 시 `punycode` 라이브러리로 검증 |
| **IDN (Internationalized Domain Name)** | 한글, 일본어 등 비ASCII 문자를 포함하는 도메인 이름 |
| **Cloudflare Workers** | Cloudflare의 글로벌 엣지 네트워크에서 실행되는 서버리스 함수. V8 isolate 기반으로 사용자에게 가장 가까운 서버에서 처리. 무료 플랜: 일 100,000건 |
| **Cloudflare KV** | Cloudflare가 제공하는 글로벌 분산 Key-Value 저장소. Workers에서 직접 호출 가능. 무료 플랜: 일 100K 읽기, 1K 쓰기, 1GB 저장 |
| **Wrangler** | Cloudflare Workers의 CLI 도구. Workers와 KV의 개발, 테스트, 배포에 사용 |
| **초성** | 한글 글자의 첫 번째 자음. 예: 가 → ㄱ, 만 → ㅁ |
| **sparse checkout** | Git 저장소에서 필요한 파일/디렉토리만 선택적으로 체크아웃하는 기능 |
| **partial clone** | Git 저장소 클론 시 blob 데이터를 지연 로딩하여 초기 클론 크기를 줄이는 기능 |

## 14. FAQ (자주 묻는 질문)

**Q: 내가 제안한 키워드가 거절당하면?**
A: PR에 반대가 더 많거나, 유효성 검증에 실패하면 거절됩니다. 거절 사유가 Issue 댓글로 안내되며, 수정 후 재제안이 가능합니다.

**Q: 투표 기간은 얼마인가요?**
A: PR 생성 후 7일간 투표를 진행합니다. 7일 이내에 임계값에 도달하면 조기 병합됩니다.

**Q: 이미 등록된 키워드의 URL을 바꾸고 싶다면?**
A: "키워드 변경 요청" Issue 양식으로 제안하면 됩니다. 기존 등록보다 높은 찬성 수가 필요합니다 (기본 임계값 × 1.5).

**Q: 영어 키워드도 등록할 수 있나요?**
A: 네, `iphone.가격.kr`처럼 영문 키워드도 지원합니다. `_en` 디렉토리에 저장됩니다.
